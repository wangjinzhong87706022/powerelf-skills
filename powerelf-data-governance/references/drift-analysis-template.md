# 设备漂移分析报告模板

用于 MAD 检测发现渐进漂移后，生成设备维护团队工单附件。

## 报告结构

```markdown
# {STCD} {站点名称}传感器渐进漂移事件分析报告

**报告编号:** DQ-{YYYY}-{STCD}-{序号}
**报告日期:** {生成日期}
**分析工具:** powerelf-data-governance MAD引擎
**报告性质:** 设备故障分析 / 维护工单附件

---

## 一、事件概述
| 项目 | 内容 |
|------|------|
| 站点编码 | {STCD} |
| 监测类型 | {类型} |
| 漂移起始 | {开始时间} |
| 漂移结束 | {结束时间} |
| 持续时间 | {小时数} |
| 采集频率 | {频率} |
| 异常记录数 | {条数} |

## 二、漂移特征
### 2.1 进入阶段 — 跳变幅度、时间
### 2.2 漂移期间 — 范围、速率(m/h)、R²、趋势方程
### 2.3 退出阶段 — 跳变幅度、时间
### 2.4 基线恢复验证 — 前后48h中位数/标准差/均值对比表

## 三、偏移量逐日分析
| 日期 | 均值 | 偏移量 | 偏移率 |

## 四、故障模式判定
### 故障类型
### 判定依据（编号列出）
### 可能原因（按概率排序）

## 五、维护建议
### 紧急（本次工单）— 现场校准、清理、检查
### 预防（后续改进）— 告警规则、过滤规则、定期校准

## 六、附件
| 文件 | 说明 |

## 七、分析方法说明
- 使用的算法和参数
```

## 关键输出文件

分析完成后应生成3个文件：
1. `{stcd}_drift_analysis_report.md` — 上述Markdown报告
2. `{stcd}_drift_report.png` — 四子图可视化（全景/趋势/偏移/变化率）
3. `{stcd}_drift_detail.csv` — 逐小时明细（含阶段标注、偏移量、变化率、备注）

## 四子图说明

1. **水位时序全景** — 蓝线实测、绿虚线基线、红区漂移区间、标注进入/退出跳变
2. **漂移期间线性趋势** — 散点+拟合线+正常波动范围阴影
3. **相对基线偏移量** — 柱状图，红=漂移期间，蓝=正常，橙虚线=告警阈值
4. **逐小时变化率** — 柱状图，红=漂移期间，蓝=正常，橙虚线=告警阈值

## CSV字段说明

| 字段 | 说明 |
|------|------|
| 阶段 | 漂移前 / 漂移期间 / 漂移后 |
| 偏移量(m) | 相对正常基线中位数的差值 |
| 偏移率(%) | 偏移量/基线 × 100% |
| 变化率(m/h) | 相邻时点水位差/时间间隔 |
| 备记 | 关键事件标注（漂移起始/恢复点等） |
