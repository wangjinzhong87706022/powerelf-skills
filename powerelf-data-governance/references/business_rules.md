# 业务规则

数据治理引擎的领域规则定义：阈值、权重、状态枚举与判定标准。

---

## 1. 异常检测阈值

### 1.1 MAD 阈值（分指标）

| 指标 | 数据表 | 检测字段 | MAD阈值 | 变化率阈值 | 说明 |
|------|--------|----------|---------|-----------|------|
| 水位 | st_rsvr_r | rz | 3.0 | 5% | 日变化平缓，异常容易识别 |
| 河道水位 | st_river_r | z | 3.0 | 5% | 同水库水位 |
| 雨量 | st_pptn_r | p | 5.0 | 不适用 | 波动大，可突变 |
| 渗压 | st_pressure_r | ext_pressure, water_pressure | 4.0 | 3% | 中等波动 |
| 渗流 | st_percolation_r | percolation | 4.0 | 3% | 同渗压 |
| GNSS位移 | dsm_dfr_srvrds_srhrds | wgs84_delta_h/x/y | 3.5 | 2% | 缓慢变化，突变即异常 |
| 流量(入库) | st_rsvr_r | inq | 4.0 | 10% | 受闸门操作影响 |
| 流量(出库) | st_rsvr_r | otq | 4.0 | 10% | 同入库流量 |
| 通用默认 | -- | -- | 4.0 | -- | 兜底值 |

### 1.2 自适应窗口参数

```
windowSize = min(max(数据量 * 0.15, 10), 50)
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 比例系数 | 0.15 | 窗口占数据量的比例 |
| 最小窗口 | 10 | 保证小样本统计意义 |
| 最大窗口 | 50 | 限制计算复杂度 |

### 1.3 综合判定置信度

| 条件 | 判定 | 置信度 |
|------|------|--------|
| MAD异常 + 变化率超标 | 确认异常 | 高 |
| MAD异常 + 变化率正常 | 可能异常 | 中 |
| MAD正常 + 变化率超标 | 可疑 | 低 |

## 2. 插值策略选择规则

### 2.1 决策条件

| 优先级 | 条件 | 策略 |
|--------|------|------|
| 1 | nullCount < 3 | 线性插值 |
| 2 | R² > 0.9 且 |a| < 0.05 | 线性插值 |
| 3 | R² > 0.7 且 |a| < 0.2 | 二次插值 |
| 4 | 周期性明显（自相关函数峰值检测） | 样条插值 |
| 5 | 其他 | 滑动平均 |

### 2.2 置信度阈值

| 阈值 | 等级 | 处理方式 |
|------|------|----------|
| >= 0.8 | 高置信度 | 直接使用，写入业务表 |
| 0.6 - 0.8 | 中置信度 | 标记"建议复核"，仍写入 |
| < 0.6 | 低置信度 | 标记"需人工复核"，不自动写入 |

### 2.3 精度要求

所有插值结果保留 3 位小数，HALF_UP 舍入。

## 3. 质量评分权重

### 3.1 维度权重

| 维度 | 权重 | 满分 | 输入指标 |
|------|------|------|----------|
| 数据质量 | 35% | 35 | 缺失率(missingRatio), 异常率(anomalyRatio) |
| 运行稳定性 | 10% | 10 | 离线天数比(offlineDateRatio), 异常天数比(anomalyDateRatio) |
| 故障频率 | 40% | 40 | 离线次数(offlineCount), 异常次数(anomalyCount) |
| 数据完整性 | 15% | 15 | 实际记录数(actualRecords), 期望记录数(expectedRecords) |

### 3.2 数据质量子权重

| 子项 | 权重 | 满分阈值 |
|------|------|----------|
| 缺失率 | 0.6 | 缺失率+异常率 >= 30% 时得 0 |
| 异常率 | 0.4 | 异常率 >= 30% 时得 0 |

### 3.3 运行稳定性子权重

| 子项 | 权重 |
|------|------|
| 离线天数比 | 0.5 |
| 异常天数比 | 0.5 |

### 3.4 故障频率惩罚

| 参数 | 值 | 说明 |
|------|-----|------|
| 每次离线扣分 | 5 | offlineCount * 5 |
| 每次异常扣分 | 5 | anomalyCount * 5 |
| 最大惩罚 | 100 | 超过则该维度得 0 分 |
| 零分阈值 | 20次 | 100 / 5 = 20 次故障 |

### 3.5 完整性分级

| 覆盖率 | 计算方式 | 说明 |
|--------|----------|------|
| >= 95% | 满分 15 | 优秀 |
| 80% - 95% | completenessRatio * 15 | 正常计算 |
| < 80% | completenessRatio * 10 | 加重惩罚 |

### 3.6 时间衰减参数

| 参数 | 值 | 说明 |
|------|-----|------|
| lambda | 0.05 | 衰减系数 |
| 当天权重 | 1.000 | 基准 |
| 7天前权重 | 0.705 | exp(-0.05*7) |
| 14天前权重 | 0.497 | exp(-0.05*14) |
| 30天前权重 | 0.223 | exp(-0.05*30) |

### 3.7 评分区间与等级

| 区间 | 等级 | 含义 | 建议 |
|------|------|------|------|
| 90-100 | 优秀 | 设备运行健康 | 无需操作 |
| 80-89 | 良好 | 轻微问题 | 持续监控 |
| 60-79 | 一般 | 需要关注 | 排查原因 |
| < 60 | 较差 | 需要维护 | 安排检修 |

### 3.8 趋势判定

| 条件 | 趋势 |
|------|------|
| 当前评分 - 上期评分 > 5 | 改善 |
| 当前评分 - 上期评分 < -5 | 恶化 |
| 其他 | 稳定 |

## 4. 缺失检测规则

### 4.1 期望周期数

```
expectedCount = ceil((endTime - startTime) / freq)
```

`freq` 来自 `eq_business_equip_relation.frequency`。

### 4.2 连续缺失告警级别

| 连续缺失周期数 | 级别 |
|---------------|------|
| 1-2 | INFO |
| 3-5 | WARNING |
| 6-10 | ERROR |
| > 10 | CRITICAL |

### 4.3 缺失模式判定

| 模式 | 条件 | 可能原因 | 建议 |
|------|------|----------|------|
| 周期性缺失 | 连续3天同时段(+/-30分钟)缺失 | 定时维护、信号遮挡 | 调整采集频率或排查环境因素 |
| 随机缺失 | 无时间规律 | 网络不稳定、设备故障 | 检查设备通信模块 |

### 4.4 缺失率预警

| 条件 | 预警 |
|------|------|
| 环比变化 > 50% | 缺失率显著上升 |
| 连续3天缺失率 > 10% | 设备数据质量持续恶化 |

### 4.5 时间段定义

| 时段ID | 名称 | 时间范围 |
|--------|------|----------|
| 1 | 凌晨 | 00:00 - 05:59 |
| 2 | 早晨 | 06:00 - 07:59 |
| 3 | 上午 | 08:00 - 10:59 |
| 4 | 中午 | 11:00 - 13:59 |
| 5 | 下午 | 14:00 - 16:59 |
| 6 | 傍晚 | 17:00 - 18:59 |
| 7 | 晚上 | 19:00 - 23:59 |

## 5. 离线检测规则

### 5.1 设备状态枚举

| 值 | 含义 | eq_equip_base.status |
|----|------|---------------------|
| 0 | 离线(offline) | status = 0 |
| 1 | 在线(online) | status = 1 |
| 2 | 异常(anomaly) | status = 2 |

### 5.2 离线阈值配置

| st_type | 站类型 | 阈值(分钟) | 采集频率(分钟) |
|---------|--------|-----------|---------------|
| SP | 水位站 | 360 | -- |
| GN | GNSS站 | 60 | 60 |
| EL | 闸门站 | 60 | 60 |
| ZS | 雨量站 | 60 | 60 |
| WQ | 水质站 | 60 | 60 |
| PP | 渗压站 | 60 | 60 |
| DP | 渗流站 | 60 | 60 |
| DD | 位移站 | 60 | 60 |
| YZ | 墒情站 | 60 | 60 |
| ZG/RR/ZQ/TT/BB/MM/SS/DC | 其他 | 0(不检测) | -- |

### 5.3 三态判定逻辑

| 条件 | 状态 |
|------|------|
| threshold == 0 | ONLINE（不检测） |
| latestTime + threshold < now | OFFLINE |
| latestTime + threshold >= now | ONLINE |
| 同设备多表状态不一致 | ERROR |

### 5.4 测站聚合规则

| 设备状态组合 | 测站状态 |
|-------------|----------|
| 任一设备 ERROR | ERROR |
| 部分 OFFLINE + 部分 ONLINE | ERROR |
| 全部 OFFLINE | OFFLINE |
| 全部 ONLINE | ONLINE |

### 5.5 渐进式告警

| 剩余比例 | 状态 | 说明 |
|----------|------|------|
| <= 0 | OFFLINE | 已离线 |
| 0 - 0.2 | WARNING | 即将离线 |
| 0.2 - 0.5 | 注意 | 数据延迟 |
| > 0.5 | 正常 | 正常运行 |

### 5.6 离线时长分级

| 离线时长 | 级别 |
|----------|------|
| 0-1 小时 | INFO |
| 1-4 小时 | WARNING |
| 4-24 小时 | ERROR |
| > 24 小时 | CRITICAL |

### 5.7 MTTR 阈值

| 参数 | 值 | 说明 |
|------|-----|------|
| 统计窗口 | 30天 | 最近30天的离线记录 |
| 预警阈值 | 4小时 | MTTR > 4h 建议维护 |

## 6. 通用业务规则

### 6.1 治理表无逻辑删除

以下 4 张治理记录表**没有 `deleted` 字段**，不做逻辑删除：
- `eq_data_anomaly_record`
- `eq_data_missing_record`
- `eq_equip_offline_record`
- `eq_equip_anomaly_record`

查询这些表时**不要添加 `deleted = 0` 条件**。

### 6.2 异常值处理流程

```
1. 将异常值设为 null
2. 调用智能插值填补
3. 合并修复数据，记录到 fixDataContent (JSON)
4. 生成异常记录: {equipmentCode, datetime, tableName, fixDataContent}
```

### 6.3 离线状态变化联动

设备状态变化时：
- 设备离线 -> 更新 `eq_equip_base.status = 0`，创建 `eq_equip_offline_record`
- 设备恢复 -> 更新 `eq_equip_base.status = 1`，更新离线记录结束时间
- 测站状态变化 -> 触发预警消息通知
