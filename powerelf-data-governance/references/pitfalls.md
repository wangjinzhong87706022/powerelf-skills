# Pitfalls & Session Notes

## skill_view 加载失败问题

**现象**：skill 位于 category 子目录 `powerelf/data-governance/` 时，`skill_view` 和 `skill_manage` 均报 "not found"。
**根因**：skill 系统期望目录名与 skill 名一致（`powerelf-data-governance/`），而非 `data-governance/`。
**已修复**：通过 `skill_manage(action='create')` 重新注册到正确路径。

## MAD=0 数据溢出

当站点水位波动极小（< 0.01m）时，MAD=0 导致 Z-score 除零。mad.py 内部已处理（返回 score=0），
但分析时需注意：
- 变化率检测中前值为0时 change_rate=0，不会触发可疑标记
- 建议设最小变化阈值（如 0.005m），低于阈值的站点标记为"数据近乎常数"而非"正常"
- 实际案例：627B9680 站点水位几乎不变，MAD 接近 0，Z-score 溢出到数十亿

## 插值策略选择：交叉验证法

选择插值策略时，不要盲选。用交叉验证对比四种策略：

1. 从正常数据中随机抽取 50 个点作为验证集
2. 把这 50 个点临时加入缺失列表
3. 用四种策略（linear/quadratic/spline/moving_avg）分别插值
4. 对比验证集的 R² 和误差，选 R² 最高的策略
5. 用最优策略修复真正的异常点

实际案例（SW003 短间隔异常）：linear R²=0.966 最优 > moving_avg 0.958 > quadratic 0.956 > spline 0.951。
水位日变化接近线性，短间隔插值 linear 效果最佳。

### ⚠️ 大缺口插值陷阱（关键）

**问题**：连续缺失超过24小时时，quadratic/spline 的 `fill_value='extrapolate'` 会在缺口边缘产生灾难性外推。

**实际案例**（SW001，144小时缺口）：
- 正常水位范围 237~239m
- quadratic 插值修复值范围 226~262m（偏差 10+米！）
- 线性插值修复值范围 237.6~238.8m（正常）
- 交叉验证 R²：linear 0.809 > quadratic 0.692 > spline 0.658

**根因**：quadratic/spline 在训练数据边界外的外推行为不可控。缺口越大，外推越远，误差越大。

**规则**：
- 缺口 ≤ 6小时：四种策略均可，交叉验证选最优
- 缺口 6~48小时：**必须用线性插值**，quadratic/spline 仅作为交叉验证参考
- 缺口 > 48小时：用「线性趋势 + 日周期修正」（见下方）

**滑动平均 NaN 问题**：当缺口长度超过滑动窗口时，MA 策略在交叉验证中产生 NaN（无法从邻域计算均值）。此时跳过 MA，只比较其余三种。

### 大缺口季节性修正策略

当连续缺失 > 48小时（如 SW001 的144小时缺口），纯线性插值虽安全但丢失日周期特征。改进方案：

```python
# 1. 线性插值基础层
trend_vals = np.interp(gap_idx, real_idx, real_vals)

# 2. 计算每小时偏移量
global_mean = np.mean(real_vals)
hourly_offset = {h: mean(所有h小时的水位) - global_mean for h in 0..23}

# 3. 计算趋势中的小时偏差
trend_mean = np.mean(trend_vals)
trend_hourly_offset = {h: mean(趋势中h小时的值) - trend_mean for h in 0..23}

# 4. 合并修正
repaired[i] = trend_vals[i] + (hourly_offset[h] - trend_hourly_offset[h])
```

效果（SW001）：
- 纯线性：衔接起点偏差 0.008m，但无日周期特征
- 季节修正：衔接起点偏差 0.018m，保留日内高低峰，缺口内 MAD 异常 0 个
- 修复范围 237.640~238.771m（完全在正常区间内）

**注意**：大缺口修复后的 R² 通常较低（SW001 为 0.81），需要在报告中明确标注「修复精度有限，建议人工复核」。

## 异常三分类模式

水库水位异常通常分三类，处理方式不同：

| 类型 | 特征 | 处理 |
|------|------|------|
| 硬错误 | 0.0, 999.99, -15.5, 整百值等传感器错误码 | 直接插值替换 |
| 渐进漂移 | 连续多小时/天偏离正常范围，进入/退出有大跳变 | 判定为传感器漂移后整体插值替换 |
| 边界异常 | MAD Z=3~5，变化率正常，集中在阶段峰谷 | 保留，属正常季节性波动 |

## 完整数据治理 Pipeline

标准执行流程（以水库水位为例）：

```
1. 确认数据源 → 用 powerelf db.py 连 127.0.0.1/powerelf_data
2. MAD 三阶段检测 → detect_anomalies + detect_change_rate + composite_judge
3. 异常分类 → 硬错误 / 渐进漂移 / 边界异常
4. 插值策略选择 → 交叉验证 4 种策略，选 R² 最高的
5. 执行修复 → 用最优策略修复硬错误和渐进漂移
6. 质量验证 → 修复后再跑 MAD，确认异常削减率
7. 导出报告 → CSV 明细 + 可视化图表
```
