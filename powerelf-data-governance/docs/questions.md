# 数据治理 Skill — 可问问题清单

> 本文档列出了 `powerelf-data-governance` skill 能回答和不能回答的问题，供 Agent 判断是否路由到此 skill。

---

## ✅ 可用此 skill 的问题

### 异常检测（MAD）

| 问题示例 | 说明 |
|---------|------|
| XX 水库最近 7 天的水位数据有没有异常？ | MAD 异常检测 |
| 帮我检测一下渗压数据的异常值 | 分指标阈值检测 |
| 这些水位数据里有异常值吗？用 MAD 算法跑一下 | MAD + 变化率综合判定 |
| 检查一下 2026-06-01 到 2026-06-07 的雨量数据是否有异常 | 指定时间窗口检测 |
| 这个异常值是传感器故障还是真实的数据波动？ | 变化率检测 + 综合判定 |

### 缺失检测

| 问题示例 | 说明 |
|---------|------|
| XX 设备昨天的数据有没有缺失？ | 期望周期数比较 |
| 检查一下 st_rsvr_r 表 128 号测站最近一天的数据完整性 | 指定表和测站 |
| 分析这段时间的缺失模式，是周期性的还是随机的？ | 缺失模式识别 |
| 连续缺失达到哪个级别了？ | INFO/WARNING/ERROR/CRITICAL 分级 |
| 最近一周缺失率趋势怎么样？ | 环比趋势分析 |

### 插值修复

| 问题示例 | 说明 |
|---------|------|
| 帮我填补一下这段缺失的水位数据 | 四策略自适应插值 |
| 用线性插值把昨天缺失的 3 个点补上 | 指定插值策略 |
| 插值结果的置信度有多高？ | 置信度评估（高/中/低） |
| 这个插值结果可以直接用吗？ | 置信度 >= 0.8 可直接使用 |
| 这种周期性的水位数据适合用什么插值方法？ | 样条插值推荐 |

### 离线检测

| 问题示例 | 说明 |
|---------|------|
| XX 设备当前是否在线？ | 三态判定（ONLINE/OFFLINE/ERROR） |
| 哪些设备离线超过 24 小时了？ | 离线时长分级 |
| 最近一个月设备的平均修复时间（MTTR）是多少？ | MTTR 计算 |
| 检查一下各设备的在线状态 | 批量检测 |
| 这个设备离线多久了？ | 时长计算 + 分级告警 |

### 质量评分

| 问题示例 | 说明 |
|---------|------|
| XX 设备的数据质量评分是多少？ | 四维度加权评分 |
| 本周数据质量排名怎么样？哪些厂商表现好？ | 厂商排名 |

### 卡滞检测

| 问题示例 | 说明 |
|---------|------|

### 极端事件区分

| 问题示例 | 说明 |
|---------|------|

### 相关性异常

## ❌ 不应问此 skill 的问题（应路由到其他 skill）

| 问题 | 应使用的 skill | 原因 |
|-----|---------------|------|
| XX 水库当前水位是多少？ | `water-situation` | 查水位不是查数据质量 |
| 展示一下最近一周的水位变化曲线 | `water-situation` | 趋势可视化，非检测 |
| 这个水位超过警戒线了吗？ | `water-warning` / `powerelf-early-warning` | 预警规则判断 |
| 今天雨量站降雨量多少？ | `water-situation` / `powerelf-chatbi` | 实时数据查询 |
| 闸门当前开度是多少？泵站是否在运行？ | `gate-pump-operation` / `powerelf-monitor` | 设备运行状态查询 |
| 查一下 XX 表的数据 | `powerelf-chatbi` | 纯数据查询，不涉及质量分析 |
| 帮我发一个预警通知 | `powerelf-early-warning` | 预警分发，非质量检测 |

---

## 关键区分原则

> **本 skill 分析"数据本身有没有异常/缺失/质量问题"，不是查询"数据内容是什么"。**

- 数据 **有没有问题** → ✅ 用 `data-governance`
- 数据 **是什么** → ❌ 用 `chatbi` / `water-situation` 等查询类 skill
- 数据 **触发了什么规则** → ❌ 用 `early-warning` 预警类 skill
