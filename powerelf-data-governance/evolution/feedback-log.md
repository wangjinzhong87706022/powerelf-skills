# 数据治理执行反馈日志

## 使用说明

当用户反馈判断不准确（漏报/误报/参数不合适）时，在此记录事件、原因分析和改进措施。
分析反馈模式后，更新 `parameters.md` 中的参数值。

## 日志格式

```
## YYYY-MM-DD 事件标题
- **事件**: 描述发生了什么
- **原因**: 分析为什么会出现这个问题
- **改进**: 采取了什么措施
- **状态**: 待调整 / 已调整 / 已验证
- **相关参数**: parameters.md 中的参数名
```
