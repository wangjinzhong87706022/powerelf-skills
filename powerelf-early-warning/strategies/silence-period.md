# 沉默期机制

## 概述

防止同一策略在短时间内重复发送通知。

## 机制

```
存储: Redis Key = "EW_NOTICE_TACTICS_KEYS_SILENT:{tacticsId}"
过期: silenceTime（单位：分钟）

判断:
  if Redis Key 存在:
    → 跳过通知（在沉默期内）
  else:
    → 正常发送通知

设置:
  通知发送完成后，写入 Redis Key，过期时间 = silenceTime
```

## 配置

| 字段 | 说明 | 单位 |
|------|------|------|
| silenceTime | 沉默时间 | 分钟 |
| enable | 是否启用 | 1/0 |

## 典型配置

| 场景 | 沉默时间 | 理由 |
|------|----------|------|
| 水位预警 | 30分钟 | 水位变化慢，30分钟内无需重复 |
| 雨量预警 | 60分钟 | 雨量持续时通知过多 |
| 大坝安全 | 15分钟 | 安全关键，缩短沉默期 |
| 设备离线 | 120分钟 | 离线状态恢复慢 |

## 与屏蔽的区别

- 沉默期：自动机制，通知发送后自动设置，到期自动解除
- 屏蔽：手动机制，需要人工设置截止时间
