# 趋势预警规则 — 新增

## 概述

检测数据的连续单调变化趋势。原始预警系统只有静态阈值比较，无法捕捉"缓慢但持续"的异常趋势。

## 核心算法

```
输入: 最近N个采集值 values[]

1. 趋势方向检测:
   上升趋势 = all(values[i] > values[i-1] for i in 1..N-1)
   下降趋势 = all(values[i] < values[i-1] for i in 1..N-1)

2. if 上升趋势 or 下降趋势:
     趋势持续次数 = N
     趋势变化量 = |values[-1] - values[0]|
     趋势变化率 = 趋势变化量 / |values[0]|
```

## 参数配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最小连续次数 | 3 | 至少连续N次同方向变化才触发 |
| 变化率阈值 | 因指标而异 | 变化率超过此值才触发 |

### 分指标参数

| 指标 | 最小连续次数 | 变化率阈值 | 说明 |
|------|-------------|-----------|------|
| 水位(rz) | 3 | 1% | 水位连续上升可能表示洪水 |
| 渗压 | 4 | 2% | 渗压连续上升可能表示渗漏 |
| GNSS位移 | 5 | 0.5% | 位移连续变化可能表示滑坡 |
| 流量 | 3 | 15% | 流量连续变化需关注 |

## 应用场景

### 水位连续上升

```
最近5次采集: 150.1, 150.3, 150.6, 150.9, 151.3
趋势: 连续上升，5次
变化率: (151.3 - 150.1) / 150.1 = 0.8%
连续次数 >= 3 ✓
变化率 < 1% → 低于阈值，不触发

最近5次采集: 150.1, 150.5, 151.0, 151.6, 152.3
趋势: 连续上升，5次
变化率: (152.3 - 150.1) / 150.1 = 1.47%
连续次数 >= 3 ✓
变化率 > 1% → 触发趋势预警
```

### GNSS位移持续偏移

```
最近6次采集: 0.1, 0.15, 0.22, 0.30, 0.39, 0.50 (mm)
趋势: 连续上升，6次
变化率: (0.50 - 0.1) / 0.1 = 400%
连续次数 >= 5 ✓
变化率 >> 0.5% → 触发趋势预警（I级）
```

## 与阈值预警的配合

```
阈值预警: 检测是否超过固定阈值（静态）
趋势预警: 检测是否持续变化（动态）

组合使用:
  1. 趋势预警可以提前发现"正在接近阈值"的情况
  2. 趋势预警可以发现"虽然未超阈值但持续恶化"的情况
  3. 两者同时触发时，取更严重的等级
```

## 预警描述

```
上升趋势: "水位连续{N}次上升，累计上涨{变化量}m（{变化率}%），当前值{当前值}m"
下降趋势: "渗压连续{N}次下降，累计下降{变化量}kPa（{变化率}%），当前值{当前值}kPa"
```
