# 状态变化预警规则

## 概述

监测值是否发生变化（Bean名: "ZGB"），当当前值与上次值不同时触发预警。

## 判断逻辑

```
输入: value(当前采集值), eqCode, dotAddress, ruleId

1. 上次值 = stateStore.get(eqCode, dotAddress, ruleId)
2. if 上次值存在 and value != 上次值:
     → 触发预警
     → 消息: "指标值从{上次值}改变为{当前值}"
3. 无论是否触发，更新 stateStore 中的值为当前值
```

原始实现使用 Redis 存储上次值，Agent 版本使用本地状态机。

## 状态存储

```
stateStore = {
  key: "{eqCode}:{dotAddress}:{ruleId}",
  value: 上次采集值,
  updateTime: 更新时间
}
```

## 规则配置格式

```json
{
  "content": [描述用min, 描述用max],
  "condition": "FOUR"
}
```

content 仅用于消息描述，实际判断只比较上次值和当前值。

## 典型应用场景

| 场景 | 说明 |
|------|------|
| 闸门操作检测 | 闸门状态从关→开或开→关 |
| 水泵启停检测 | 水泵从停止→运行或反之 |
| 报警状态变化 | 报警器从正常→报警 |
| 设备模式切换 | 设备从自动→手动或反之 |

## 预警描述

```
"已触发【状态变化预警】: 指标值从{旧值}改变为{新值}"
```

## 与开关量预警的区别

- 开关量预警(KG)：值等于预设的开/关值时触发
- 状态变化预警(ZGB)：值与上次不同时触发

两者可以配合使用：KG 检测绝对状态，ZGB 检测状态变化。
