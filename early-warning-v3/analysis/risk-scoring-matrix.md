# 风险评分矩阵

## 概述

综合多维度因素评估告警风险等级，为响应决策提供量化依据。

## 评分因素

| 因素 | 权重 | 评分规则 |
|------|------|----------|
| 告警级别 | 30% | 红(1)=4, 橙(2)=3, 黄(3)=2, 蓝(4)=1 |
| 告警数量 | 20% | ≥5条=4, 3-4条=3, 2条=2, 1条=1, 0条=跳过评估 |
| 跨域关联 | 25% | 三域关联=4, 双域关联=3, 单域=2 |
| 持续时间 | 15% | >4h=4, 2-4h=3, 1-2h=2, <1h=1 |
| 趋势 | 10% | 恶化=4, 持平=3, 改善=2 |

> **有效得分范围：1.0 - 4.0**

## 前置条件

在执行评分计算前，必须先检查活跃告警数量：

- **0 条活跃告警** → 跳过风险评估，直接返回"当前无告警，无需评估风险"
- **≥ 1 条活跃告警** → 执行完整评分流程

## 评分计算

```
综合得分 = Σ(因素得分 × 权重)
```

示例：
- 告警级别=红(4) × 0.30 = 1.20
- 告警数量=3-4条(3) × 0.20 = 0.60
- 跨域关联=三域(4) × 0.25 = 1.00
- 持续时间=2-4h(3) × 0.15 = 0.45
- 趋势=恶化(4) × 0.10 = 0.40
- 综合得分 = 1.20 + 0.60 + 1.00 + 0.45 + 0.40 = 3.65 → 高风险

## 风险等级划分

| 综合得分 | 风险等级 | 响应要求 |
|----------|----------|----------|
| ≥ 3.5 | 高风险 (high) | 立即响应，建议触发预案 |
| 2.5 - 3.49 | 中风险 (medium) | 加强监控，准备预案 |
| < 2.5 | 低风险 (low) | 持续监控 |

## 特殊规则

### 规则执行优先级

特殊规则按以下优先级顺序执行，前序规则命中后不再执行后续规则：

| 优先级 | 规则类型 | 说明 |
|--------|----------|------|
| 1 | 直接判定规则 | 命中即跳过评分计算，直接输出风险等级 |
| 2 | 降级规则 | 在评分计算完成后应用，上限不超过中风险 |
| 3 | 加权评分计算 | 正常评分流程 |
| 4 | 边界提升规则 | 对加权评分结果的微调，仅在 3.40 - 3.49 区间生效 |

### 直接判定规则（跳过评分）

1. **红色告警 + 三域关联** → 直接判定高风险
   - 条件：存在 level_r=1 的告警，且同时存在水位、降雨、渗流三类告警

2. **告警风暴** → 直接判定高风险
   - 条件：1小时内告警数 > 50 条

### 降级规则

3. **所有告警已确认** → 最高判定中风险
   - 条件：所有活跃告警的 message_confirm = 1

### 边界提升规则

4. **评分边界值** → 上浮为高风险
   - 条件：综合得分在 **3.40 - 3.49** 之间（注意：≥ 3.50 已由风险等级表直接判定为高风险，此处仅覆盖 3.40 - 3.49 的区间）
   - 理由：宁可误报不可漏报

## 各因素评分细则

### 告警级别评分

取所有活跃告警中的最高级别：

| level_r | 级别名称 | 得分 |
|---------|----------|------|
| 1 | 特别严重（红色） | 4 |
| 2 | 严重（橙色） | 3 |
| 3 | 较重（黄色） | 2 |
| 4 | 一般（蓝色） | 1 |

### 告警数量评分

统计 message_confirm=0 且 deleted=0 的活跃告警数：

| 数量 | 得分 |
|------|------|
| 0 条 | 跳过风险评估，返回"当前无告警"（见前置条件） |
| 1 条 | 1 |
| 2 条 | 2 |
| 3-4 条 | 3 |
| ≥ 5 条 | 4 |

### 跨域关联评分

基于关联分析结果中的 compound_risks。跨域分析将告警归为三大"域"：

| 域 | 涵盖的 ew_type | 说明 |
|----|----------------|------|
| 水位域 | 0 (水位) | 库水位、坝前水位等 |
| 降雨域 | 2 (雨量) | 降雨量告警 |
| 渗流域 | 20 (渗压/流量)、40 (渗流/位移) | SKILL.md 中 ew_type=20 为"渗压/流量"，ew_type=40 为"渗流/位移"，两者均属大坝渗流安全监测范畴，在跨域关联分析中合并为同一"渗流域" |

> **仅考虑水位(0)、降雨(2)、渗流(20/40)三域的原因：** 这三类告警构成水库安全的"水文-结构"因果链（降雨 → 水位上涨 → 坝体渗流压力增大），是风险关联分析的核心维度。其他类型（如水质=1、设备离线=12）属于运维层面，不纳入跨域风险评分。

| 关联类型 | 得分 | 判断标准 |
|----------|------|----------|
| 三域关联 | 4 | 同时存在水位(ew_type=0) + 降雨(ew_type=2) + 渗流(ew_type=20或40) 告警 |
| 双域关联 | 3 | 存在任意两类告警 |
| 单域 | 2 | 仅存在单一类型告警 |

### 持续时间评分

计算最早活跃告警的持续时间：

| 持续时间 | 得分 |
|----------|------|
| > 4 小时 | 4 |
| 2 - 4 小时 | 3 |
| 1 - 2 小时 | 2 |
| < 1 小时 | 1 |

### 趋势评分

趋势判断采用**两级指标**体系：

| 优先级 | 指标 | 数据来源 | 说明 |
|--------|------|----------|------|
| PRIMARY（主要指标） | 水位变化趋势 | st_rsvr_r 表最近 N 条记录的 rz 值 | 水位是水库安全的核心物理量，优先使用 |
| SECONDARY（辅助指标） | 告警数量变化趋势 | ew_info_message 表的时间序列统计 | 仅在水位数据缺失（无测站数据或 rz 全为 NULL）时作为替代 |

> **指标选择逻辑：** 水位数据可用时，以水位趋势为准；水位数据不可用时，退化为以告警数量变化趋势为准。两者不混合使用。

| 趋势 | 得分 | 判断标准 |
|------|------|----------|
| 恶化 | 4 | [PRIMARY] 水位持续上涨；[SECONDARY] 告警数量持续增加 |
| 持平 | 3 | [PRIMARY] 水位稳定（波动 < 5%）；[SECONDARY] 告警数量不变 |
| 改善 | 2 | [PRIMARY] 水位下降；[SECONDARY] 告警数量持续减少 |

## 边界场景处理

### 场景1: 仅有蓝色告警
- **输入**: 3条蓝色告警（level_r=4），无跨域关联
- **计算**: 级别=1, 数量=3, 关联=2, 持续=1, 趋势=3
- **得分**: 1×0.3 + 3×0.2 + 2×0.25 + 1×0.15 + 3×0.1 = 0.3+0.6+0.5+0.15+0.3 = 1.85
- **结果**: 低风险（<2.5）

### 场景2: 单条红色告警无关联
- **输入**: 1条红色告警（level_r=1），无跨域关联
- **计算**: 级别=4, 数量=1, 关联=2, 持续=1, 趋势=3
- **得分**: 4×0.3 + 1×0.2 + 2×0.25 + 1×0.15 + 3×0.1 = 1.2+0.2+0.5+0.15+0.3 = 2.35
- **结果**: 低风险（<2.5）

### 场景3: 边界值3.45
- **输入**: 综合评分为3.45
- **规则**: 3.40-3.49范围，应用边界提升规则
- **结果**: 高风险（上浮）

### 场景4: 所有告警已确认
- **输入**: 5条告警，全部message_confirm=1
- **规则**: 应用降级规则，最高判定中风险
- **结果**: 中风险（即使评分>3.5）

### 场景5: 红色告警+三域关联（直接判定）
- **输入**: 1条红色水位告警 + 1条橙色降雨告警 + 1条黄色渗流告警
- **规则**: 满足直接判定条件，跳过评分计算
- **结果**: 高风险（直接判定）

## Agent 行为指引

执行风险评估时：
1. 收集所有活跃告警数据
2. 检查前置条件（0条告警→跳过评估）
3. 检查是否满足直接判定规则（优先级1）
4. 检查是否满足降级规则（优先级2）
5. 计算各因素得分
6. 计算综合得分（加权评分）
7. 检查边界提升规则（优先级4）
8. 输出风险等级和各因素详情
