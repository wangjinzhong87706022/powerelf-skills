# 巡检质量评分模型

4 维度加权评分体系，满分 100 分。

## 评分维度

| 维度 | 权重 | 数据来源 |
|------|------|---------|
| 完成率 | 30分 | `business_check_task.real_objitem / plan_objitem` |
| 及时率 | 25分 | `business_check_task.exceed_time` |
| 缺陷发现率 | 25分 | `bad_num / real_objitem` |
| 路线覆盖率 | 20分 | `check_percent` |

## 评分标准（分段式）

### 完成率（30分）

| 区间 | 得分 | 说明 |
|------|------|------|
| ≥95% | 30 | 优秀 |
| [90%, 95%) | 25 | 良好 |
| [80%, 90%) | 20 | 一般 |
| [70%, 80%) | 15 | 较差 |
| <70% | 10 | 不合格 |

### 及时率（25分）

| 区间 | 得分 |
|------|------|
| ≥95% | 25 |
| [90%, 95%) | 20 |
| [80%, 90%) | 15 |
| <80% | 10 |

### 缺陷发现率（25分）

| 区间 | 得分 | 含义 |
|------|------|------|
| [1%, 5%) | 25 | 最佳区间 — 设备维护良好<br>+ 检查质量高 |
| [5%, 10%) | 20 | 设备老化 |
| [10%, 20%) | 15 | 设备较差 |
| ≥20% | 10 | 设备严重老化 |
| <1% | 15 | 可能漏检 |

### 路线覆盖率（20分）

| 区间 | 得分 |
|------|------|
| ≥95% | 20 |
| [90%, 95%) | 15 |
| [80%, 90%) | 10 |
| <80% | 5 |

## 等级划分

| 等级 | 总分 | 含义 |
|------|------|------|
| A | [90, 100] | 优秀 |
| B | [80, 89] | 良好 |
| C | [70, 79] | 一般 |
| D | [60, 69] | 较差 |
| E | [0, 60) | 不合格 |

## 异常规则

| 规则 | 条件 | 级别 |
|------|------|------|
| 超时预警 | 及时率<80% | WARNING |
| 漏检预警 | 遗漏率>20% | WARNING |
| 缺陷积压 | 缺陷status=0 >7天 | CRITICAL |
| 连续缺陷 | 同一obj_id 3次以上 | WARNING |

## Python 实现

```python
def compute_score(value, breakpoints):
    """分段评分：breakpoints = [(95, 30), (90, 25), (80, 20), (70, 15), (0, 10)]"""
    for threshold, score in breakpoints:
        if value >= threshold:
            return score
    return breakpoints[-1][1]
```
