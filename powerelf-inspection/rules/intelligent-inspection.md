# 智能巡检规则

## 概述

智能巡检 = Agent自动采集分析 + 人工现场复核。Agent负责数据密集型工作，人工负责Agent无法覆盖的感官检查。

## 数据源映射（基于实际数据库表）

### 测站关联

| 测站表 | 字段 | 类型说明 |
|--------|------|----------|
| `att_st_base` | `id`, `code`(站码char18), `name`(站名), `longitude`(经度), `latitude`(纬度), `type`(站类型字典), `status`(0=离线/1=在线/2=异常) | 19+种站类型 |
| `dsm_cz_info` | `id`, `code`(测点编号), `name`, `x`(坐标X), `y`(坐标Y), `type`(1=测站点/2=后视点) | GNSS测点信息 |

巡检点坐标(`business_check_point.lon_lat`)与测站坐标(`att_st_base.longitude/latitude`)通过Haversine距离匹配。

| 分析项目 | 分析方法 | 规则文件 | 替代的人工操作 |

| 缺陷预测 | 历史趋势回归 | defect-prediction.md | 无（纯新增能力） |
| 巡检质量评分 | 加权评分模型 | quality-assessment.md | 人工统计考核 |
| 多指标关联分析 | 相关性分析 | ../../_shared/rules/reservoir-analysis.md | 人工综合多张报表 |

### 人工巡检（Agent无法替代）

| 巡检项目 | 需要人工的原因 | 建议频率 | 可优化方向 |
|----------|---------------|---------|-----------|
| 混凝土裂缝检查 | 需要近距离目测+触摸 | 月度 | 高清摄像头+裂缝AI识别 |
| 钢结构锈蚀检查 | 需要多角度观察 | 季度 | 摄像头定期拍照+AI比对 |
| 渗漏点确认 | 需要到现场看水流痕迹 | 异常触发 | 渗流传感器报警后人工确认 |
| 闸门振动/异响 | 需要听觉/触觉 | 周度 | 振动传感器+声学分析 |
| 电气设备烧焦味 | 需要嗅觉 | 周度 | 烟雾/气体传感器 |
| 水下结构检查 | 需要潜水或水下机器人 | 年度 | 水下ROV+声呐 |
| 启闭机润滑检查 | 需要打开设备观察 | 月度 | 暂无替代 |
| 防汛物资清点 | 需要实物清点 | 季度 | RFID标签+自动盘点 |

## 智能巡检工作流

### 日常巡检流程（非异常日）

```
┌──────────────────────────────────────────────────────────┐
│  每日自动执行（Agent，无人值守）                            │
│                                                           │
│  06:00  采集昨日全天监测数据                                │
│  06:05  数据质量检查（缺失/异常/离线）                      │
│  06:10  异常检测（MAD + 趋势分析）                          │
│  06:15  生成巡检日报                                        │
│  06:20  推送日报到相关负责人                                 │
│                                                           │
│  无异常 → 流程结束，不需要人工介入                           │
│  有异常 → 进入异常处理流程                                  │
└──────────────────────────────────────────────────────────┘
```

### 异常处理流程

```
┌──────────────────────────────────────────────────────────┐
│  Agent发现异常                                             │
│  ├── 1. 自动截图/录像（调用摄像头API）                      │
│  ├── 2. 自动调取相关历史数据                                │
│  ├── 3. 自动分析异常原因（规则引擎）                        │
│  ├── 4. 自动评估风险等级                                    │
│  └── 5. 推送预警通知（短信/APP/微信）                       │
└──────────────────────────────────────────────────────────┘
           ↓
┌──────────────────────────────────────────────────────────┐
│  巡检员收到通知                                            │
│  ├── 1. 查看Agent提供的异常分析报告                         │
│  ├── 2. 到现场复核确认                                     │
│  ├── 3. 拍照上传（补充Agent无法获取的信息）                  │
│  ├── 4. 填写处理意见                                       │
│  └── 5. 提交复核结果                                       │
└──────────────────────────────────────────────────────────┘
           ↓
┌──────────────────────────────────────────────────────────┐
│  Agent自动处理复核结果                                      │
│  ├── 1. 更新缺陷记录                                       │
│  ├── 2. 如果确认严重 → 自动启动应急预案流程                  │
│  ├── 3. 记录人工反馈（用于模型优化）                         │
│  └── 4. 生成巡检报告                                       │
└──────────────────────────────────────────────────────────┘
```

### 定期巡检流程（月度/季度）

```
┌──────────────────────────────────────────────────────────┐
│  Agent巡前准备（巡检前1天）                                 │
│  ├── 1. 分析本月监测数据趋势                                │
│  ├── 2. 标记需要重点关注的巡检点                             │
│  ├── 3. 生成巡检建议清单                                    │
│  └── 4. 推送到巡检员手机APP                                 │
└──────────────────────────────────────────────────────────┘
           ↓
┌──────────────────────────────────────────────────────────┐
│  人工现场巡检                                              │
│  ├── 1. 按Agent建议的重点清单巡检                           │
│  ├── 2. 补充Agent无法检测的项目（裂缝/锈蚀/异响）           │
│  ├── 3. 手机APP现场记录+拍照                               │
│  └── 4. 签到打卡（GPS/二维码/RFID）                        │
└──────────────────────────────────────────────────────────┘
           ↓
┌──────────────────────────────────────────────────────────┐
│  Agent巡后处理                                             │
│  ├── 1. 合并Agent数据 + 人工巡检数据                        │
│  ├── 2. 自动生成巡检报告（含数据图表+人工照片）              │
│  ├── 3. 评估巡检质量（遗漏率/及时率）                       │
│  └── 4. 归档+推送                                         │
└──────────────────────────────────────────────────────────┘
```

## 摄像头分析能力

### 已实现：报警状态监控

通过 `ew_camera_info` 表监控摄像头报警状态：
- `alarm_code`: 报警码
- `alarm_stat`: 报警状态（1=产生, 2=消失）
- `confirm`: 确认状态

Agent 可查询当前报警状态，推送通知给巡检员现场确认。

### 规划中：AI图像识别（需集成AI模型）

| 识别项目 | 技术方案 | 精度预期 | 替代的人工操作 | 状态 |
|----------|----------|----------|---------------|------|
| 水位标尺读数 | OCR文字识别 | 高(95%+) | 人工读水尺 | 规划中 |
| 水面漂浮物 | 目标检测(YOLO) | 中(80%+) | 人工巡查水面 | 规划中 |
| 人员闯入 | 人体检测 | 高(95%+) | 人工安保巡逻 | 规划中 |
| 大面积渗水 | 图像分割 | 中(75%+) | 人工目测坝面 | 规划中 |
| 闸门开度 | 刻度线检测 | 高(90%+) | 人工看刻度 | 规划中 |
| 设备外观变化 | 图像比对(diff) | 中(80%+) | 人工拍照对比 | 规划中 |
| 烟雾/火焰 | 烟火检测模型 | 高(90%+) | 人工发现 | 规划中 |

### 图像比对流程

```
1. 定时截图（每小时/每天）
2. 与基准图片对比（首次安装时拍摄的标准状态）
3. 检测差异区域
4. 差异超过阈值 → 推送人工确认
5. 人工确认后更新基准图片
```

## 报告自动生成

### Agent自动填充内容

| 报告章节 | 数据来源 | 自动化程度 |
|----------|----------|-----------|
| 水位/雨量/流量数据 | 传感器API | 100%自动 |
| 位移/渗压趋势 | GNSS/渗压计API | 100%自动 |
| 设备运行状态 | 设备状态API | 100%自动 |
| 数据质量统计 | 数据治理API | 100%自动 |
| 预警情况汇总 | 预警系统API | 100%自动 |
| 异常分析说明 | Agent规则引擎 | 90%自动(需人工审核) |
| 现场照片 | 摄像头截图 + 人工拍照 | 部分自动 |
| 处理建议 | Agent规则引擎 | 80%自动(需人工确认) |
| 签字盖章 | — | 纯人工 |
