# 路线优化规则

## 优化目标

- 降低遗漏率（< 10%）
- 减少超时率（< 5%）
- 提高缺陷发现率

## 问题诊断

### 1. 遗漏率过高

```
诊断条件：路线平均遗漏率 > 20%

可能原因：
  a) 路线包含过多巡检点 → 建议拆分路线
  b) 巡检时间不足 → 建议延长 maxTime
  c) 巡检点位置分散 → 建议按地理区域重新排序
  d) 检查项过多 → 建议精简非必要检查项

优化建议：
  if 巡检点 > 10个:
    建议拆分为2条路线
  if 单点平均时间 > 30分钟:
    建议精简检查项
```

### 2. 频繁超时

```
诊断条件：路线超时任务占比 > 30%

可能原因：
  a) maxTime 设置过短
  b) 巡检点距离过远
  c) 检查项过于复杂
  d) 执行人不熟悉路线

优化建议：
  计算实际平均用时 = AVG(endTime - beginTime)
  if 实际用时 > maxTime * 1.2:
    建议延长 maxTime 至实际用时的 1.1 倍
  else:
    建议优化路线顺序或减少检查项
```

### 3. 缺陷发现率低

```
诊断条件：路线缺陷发现率 < 1%

可能原因：
  a) 设备状态良好（正常）
  b) 巡检标准过低
  c) 执行人未认真检查（漏检）

判定方法：
  if 周边路线缺陷率也低:
    → 该区域设备状态良好
  else:
    → 可能是执行问题，建议轮换执行人
```

## 输出报告格式

```markdown
## 路线优化建议

### 路线1：大坝巡查
- 当前状态：遗漏率25%，超时率40%
- 问题：12个巡检点，maxTime=2小时，实际平均用时3.5小时
- 建议：
  1. 拆分为"大坝上游"和"大坝下游"两条路线
  2. 新路线maxTime设为2小时
  3. 按地理顺序重新排列巡检点

### 路线2：闸门检查
- 当前状态：遗漏率5%，超时率2%，缺陷发现率8%
- 评价：运行良好，无需调整
```
