# 报告交付前 QA 闸（Pre-Delivery QA Checklist）

> **何时过**：报告组装完毕、交付前（在 `statistical-caution.md` **之前**）。
> **性质**：被动清单 + Agent 自判，不做代码级强制，不自动打分。
> **姊妹关系**：QA 闸查"数字和逻辑站不站得住"；`statistical-caution.md` 查"措辞和结论过不过"；两道先后过。
> **来源**：复用 `knowledge-work-plugins/data` 的 `validate-data` skill 方法论，适配水利场景。

---

## 一、三级置信度评级

报告末尾由 Agent 根据本清单自判，填入 `confidence_tier` 字段：

| 等级 | 含义 | 适用场景 |
|------|------|----------|
| **Ready to share** | 数据完整、计算正确、数字自洽、异常已标注 | 核心指标无缺失、QA 清单全绿 |
| **Share with caveats** | 有已知局限但不影响结论主体 | 部分数据缺失（已标注）、存在合理近似（已说明） |
| **Needs revision** | 存在影响结论的缺陷，需修正后交付 | 关键指标缺失/错误、计算错误、逻辑不自洽 |

**评定时机**：报告组装完毕、交付前。不自动判定——Agent 逐条核对后填入。

---

## 二、四类 QA 清单

### 2.1 数据质量

- [ ] **数据新鲜度**：报告覆盖时段的最新数据是否已入库？最大时间戳与"报告生成时间"的间隔是否在预期内？（预期：日报 ≤1 小时，旬报 ≤1 天）
- [ ] **缺失值处理**：空值率是否在 `completeness_tier` 定义的阈值内？若超过阈值，是否已标注并说明影响？
- [ ] **去重逻辑**：若数据存在重复（同站同时间多条），是否已按规则去重（取最新/平均/合并）？去重规则是否在报告或注释中说明？
- [ ] **过滤标注**：是否剔除了已知不合格数据（如设备离线、通信中断）？若未剔除，是否在报告中明确标注数据包含离线时段？

**水利映射**：
- 水库水情表（`st_rsvr_r`）的 `inq`/`otq` 流量数据常含缺失，检查去重是否按 `st_id + tm` 复合键。
- 雨量表（`st_pptn_r`）存在停采期（冬季停测），检查缺失是否被误判为"无降雨"。

---

### 2.2 计算

- [ ] **聚合粒度**：时间聚合（小时/日/旬/月）是否匹配报告周期？若报告中混合了不同粒度，是否已说明？
- [ ] **分母正确性**：计算异常率、达标率、环比等比率指标时，分母是否为"应在线设备数"而非"实际有数据设备数"？（离线设备应计入分母，否则分母被放大导致异常率虚低）
- [ ] **日期对齐**：比较不同时段（如本月 vs 上月、今年 vs 去年）时，是否对齐了日历结构？（本月 31 天 vs 上月 28 天，日环比需考虑天数差异）
- [ ] **JOIN 多对多**：若涉及多表 JOIN（如测点-设备-厂站），是否验证了 JOIN 结果行数不超过预期？（多对多 JOIN 会指数级膨胀行数）
- [ ] **指标定义一致**：报告中同一指标（如"异常率"）的定义是否在全篇保持一致？前置章节与结论章节是否用了同一公式？
- [ ] **小计 = 总计**：若有分区/分站小计，是否验证了小计之和等于总计？

**水利映射**：
- 大坝安全监测常涉及"测点-测线-坝段-坝区"多级分组，JOIN 时需防止测点重复。
- 跨表 JOIN 如 `st_pressure_r` 与 `st_rsvr_r`，检查时间戳字段名是否一致（`tm` vs `data_time`）。

---

### 2.3 合理性

- [ ] **量级 smell test**：关键指标是否在合理范围内？（如水位应在死水位与正常蓄水位之间，雨量日累计通常 <200mm，渗压 kPa 量级应与库水位正相关）
- [ ] **趋势连续**：相邻时段/相邻测站的指标变化是否连续？若有跳变，是否有合理解释（设备更换、工况变化、极端事件）？
- [ ] **交叉引用自洽**：报告中不同章节引用的同一数值是否一致？（如摘要的"平均水位"与正文表格的"月均值"是否匹配）
- [ ] **边界条件**：最小/最大/零值是否合理？（如流量不能为负、雨量不能为负、水位不能突变 ±10m）

**水利映射**：
- 量级基准参考 `_shared/references/mad.md` 阈值表做 smell test。
- 闸门开度变化应平滑（步进式控制），若出现跳变检查是否编码错误。

---

### 2.4 呈现

- [ ] **零基坐标**：折线/柱状图纵轴是否从零开始（或标注了截断）？截断图应显式标注"非零基"。
- [ ] **数字格式**：同一指标的精度是否一致？（如"水位：12.34 m" vs "水位：12.3 m" 混用）
- [ ] **Caveat 透明**：是否标注了数据质量 caveat？（如"本月含 15% 缺失值""某测站 3 月 1 日起停测""此处采用插值结果，置信度见数据画像"）
- [ ] **可复现**：是否提供了数据时间窗口、来源表、关键参数（如离群检测方法/阈值）？

## 三、8 类陷阱（水利映射）

| 陷阱 | 典型场景 | 自检问句 |
|------|----------|----------|
| **JOIN 爆炸** | 测点-设备-厂站多对多 JOIN | JOIN 结果行数是否在预期内？ |
| **时区错配** | 多源时间戳（采集时间/入库时间/上报时间）混用 | 报告中时间列来自哪张表的哪个字段？ |
| **分母漂移** | 在线设备数变化时计算异常率/达标率 | 分子分母是否来自同一时间点？ |
| **不完整周期** | 汛期/日比较（本月 vs 上月） | 两时段日历结构是否一致？ |
| **平均的平均** | 先按站算均值，再对各站均值取平均 | 是否对各站加权？权重是否合理？ |
| **选择偏差** | 仅分析在线设备 | 离线设备是否应纳入分母？ |
| **幸存者偏差** | 只分析当前存活的测站 | 已退役/搬迁测站是否影响趋势？ |
| **插值未标注** | 缺失值插值后直接参与统计 | 插值点是否已标注？置信度如何？ |

## 四、红旗量级（快速参考）

| 量级类型 | 红旗阈值 | 说明 |
|----------|----------|------|
| **完整性 tier** | 红 <80% / 橙 80-95% / 黄 95-99% / 绿 >99% | 见 `lib/profiling.py:completeness_tier()` |
| **空值率** | >30% 需说明原因 | 超过此阈值时报告中必须标注 |
| **环比突变** | | ±50% 且绝对值 >2σ | 需排查数据或工况变化 |
| **时间跨度不匹配** | 前后章节引用时段不同 | 导致"异常率"无意义 |
| **SOGI/FAM 指标** | 无数据或计算错误 | 出现即标记为 `[FAM]` / `[SOGI]` |

## 五、报告组装自检流程

```
1. 完成报告正文（含摘要、数据画像、分析结论）
       ↓
2. 过一遍第二节「四类 QA 清单」（数据质量→计算→合理性→呈现）
3. 对照第三节「8 类陷阱」逐条检查
4. 根据检查结果自判 confidence_tier
       ↓
5. 报告末尾追加「QA 自检节」模板（checkbox 形式，由 Agent 逐条填 √/✗/说明）
6. 挂 confidence_tier 字段（Ready to share / Share with caveats / Needs revision）
       ↓
7. 过 statistical-caution.md（措辞护栏）
8. 最终交付
```

> 数据画像方法论见 [`../data-profiling.md`](../data-profiling.md)，统计措辞护栏见 [`../statistical-caution.md`](../statistical-caution.md)。
